## How to Contribute

### Find an Issue or Feature Request
- Check the [Issues](https://github.com/justinlake/example-components/issues) tab for open issues or feature requests.
- Feel free to suggest new features or report bugs.

### Fork the Repository
- Fork this repository to your own GitHub account.
- Clone your fork locally:
  ```bash
  git clone https://github.com/yourusername/example-components.git
  ```

### Create a New Branch
- Use descriptive branch names:
  ```bash
  git checkout -b feature/your-feature-name
  ```

### Make Your Changes
- Implement your feature or bug fix.
- Write clear, concise, and well-documented code.
- Add or update tests as appropriate.

### Run Tests (if applicable)
- Ensure all existing and new tests pass:
  ```bash
  npm test
  ```
  or
  ```bash
  yarn test
  ```

### Commit Your Changes
- Follow [conventional commits](https://www.conventionalcommits.org/) or use meaningful commit messages.
  ```bash
  git add .
  git commit -m "feat: Add feature to do X"
  ```

### Push and Submit a Pull Request
- Push your branch:
  ```bash
  git push origin feature/your-feature-name
  ```
- Open a [pull request (PR)](https://github.com/justinlake/example-components/pulls) describing your changes.
